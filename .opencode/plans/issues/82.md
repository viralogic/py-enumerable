# Fix #82: Unexpected IndexError when iterable is partially consumed — Result

## Root Cause Analysis

### The Bug

`Enumerable.first(func)` on a large collection (256+ items) intermittently raises `IndexError`, and `len()` reports fewer items than expected.

### Execution Flow

```python
ACI = Enumerable([...])     # 256 dicts
ACI.first(lambda c: c["aci"] == 100)
```

1. `first(func)` → `self.where(func).element_at(0)` → `Enumerable(filter(predicate, self))`
2. `element_at(0)` → `self[0]` → `for i, e in enumerate(self): if i == n: return e`
3. Iterating the filtered Enumerable means iterating `filter(predicate, ACI._iterable)`
4. `filter` lazily pulls from `RepeatableIterable.__iter__()` (fresh generator each call)
5. In `RepeatableIterable.__iter__()`, when `_root is None`, it builds the linked-list **while yielding**:
   ```python
   for index, item in enumerate(self._data):
       node = Node(value=item)
       if index == 0:
           self._root = node
           ...
       yield node.value        # <-- yields mid-build
   self._tail = self._current  # <-- never reached if interrupted
   self._len = i               # <-- never reached if interrupted
   ```
6. When the predicate matches (aci=100 at index 99), `__getitem__` does `return e`, which terminates the generator chain. `RepeatableIterable.__iter__()` receives `GeneratorExit` and **stops without setting `_tail` or `_len`**.
7. **Result**: Linked list is cached with only 100 nodes, `_tail` is `None`, `_len` is `None`.
8. Later, `len(ACI)` → `RepeatableIterable.__len__()`:
   - `self._len is None` → `sum(1 for item in iter(self))`
   - `self._root is not None` → walks from `_root` following `next` pointers
   - Only 100 nodes exist → returns 100 instead of 256
9. `ACI.first(lambda c: c["aci"] == 256)` → `element_at(0)` → `__getitem__` iterates 100 nodes, never finds a match → returns `None` → `raise IndexError`

## Chosen Design / Fix

**Strategy**: Keep lazy linked-list building (no eager caching), but fix re-entry after early termination by detecting whether the data source is reusable.

### The three-case approach in `__iter__()`

1. **First access (`_root is None`)**: Lazily build linked list as items are yielded, same as before. No change.

2. **Re-entry with partial cache + reusable source** (e.g., list, range — has no `__next__`): Reset `_root`, `_tail`, `_len`, `_current` to `None` and re-iterate from scratch. The source is repeatable, so this is correct.

3. **Re-entry with partial cache + one-shot iterator** (e.g., generator, filter — has `__next__`): Iterate existing cached nodes first, then resume building from the partially-consumed source. The iterator continues from where it left off, so the cache is extended correctly.

### Why not eager caching

Eager caching (building the full linked list before yielding) would make `first()` O(N) instead of O(1) for large collections, violating LINQ-like semantics where terminal operations short-circuit.

### `__reversed__()` fix

When `_tail` is `None` (cache incomplete), traverse from `_root` to find the tail instead of yielding nothing.

## Changes Applied

### 1. `py_linq/core.py` — `RepeatableIterable.__iter__()` (origin:101-140)

Added entry guard before the main `if/else`:
```python
if self._root is not None and self._len is None and not hasattr(self._data, '__next__'):
    self._root = None
    self._current = None
    self._tail = None
    self._len = None
```

This resets the cache when re-entering after early termination on a reusable source (list/range/etc.).

In the re-entry (`else`) branch, changed resume logic to use `last_node` (tracked during cache iteration) instead of `_tail`:
```python
last_node = None
while self._current is not None:
    yield self._current.value
    last_node = self._current
    self._current = self._current.next
    i += 1
if self._len is None:
    prev_node = last_node
    for item in self._data:
        ...
```

### 2. `py_linq/core.py` — `RepeatableIterable.__reversed__()` (origin:90-99)

Added fallback to find the tail node when cache is partial:
```python
tail = self._tail
if tail is None and self._root is not None:
    cur = self._root
    while cur.next is not None:
        cur = cur.next
    tail = cur
current = tail
```
